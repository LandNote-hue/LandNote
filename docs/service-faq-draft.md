## A. 역할·가입 경로 (CEO vs 직원 vs 개인)

### FAQ-A1 · 회사명을 입력했는데 직원(Member)으로 가입됐어요

- **증상:** 설정 → 계정 유형에 「직원」 표시, 멤버 관리 메뉴 없음
- **원인:** **초대 링크**(`/signup/invite?token=...`)로 가입. 화면의 회사명은 입력칸이 아니라 초대된 회사 **안내 문구**
- **해결:** 대표는 `/signup` → 「회사·팀으로 시작」+ 회사명. 이미 직원으로 가입했다면 탈퇴 후 재가입 또는 관리자 SQL 복구 (`supabase/scripts/repair_ceo_account.sql`)

### FAQ-A2 · 대표(CEO)로 가입하려면?

- **해결:** `/signup` → 「**회사·팀으로 시작**」→ **회사명 필수** → 가입. 초대 링크로는 CEO 가입 불가

### FAQ-A3 · 직원은 어떻게 가입하나요?

- **해결:** 대표가 멤버 관리에서 만든 `/signup/invite?token=...` 링크로만 가입

### FAQ-A4 · 개인(Solo)과 회사 대표 차이

- **개인:** 「개인으로 시작」— 팀·멤버 관리 없음
- **대표:** 「회사·팀으로 시작」— 멤버 초대·권한 관리

### FAQ-A5 · 구글 로그인 후 개인(Solo)으로 가입됐어요

- **원인:** 구글 로그인 후 가입 완료 화면 기본값이 「개인으로 시작」
- **해결:** 가입 완료 단계에서 「회사·팀으로 시작」+ 회사명 입력

### FAQ-A6 · 대표인데 멤버 관리 메뉴가 없어요

- **원인:** 직원/Solo 계정이거나 Supabase 미연결
- **해결:** 설정 → **계정 유형** 확인. 「대표(CEO)」일 때만 표시. `/team/manage` 또는 설정 → 「멤버 관리 열기」

## B. 초대 링크

### FAQ-B1 · 초대 링크 만료

- **증상:** 「초대 링크를 사용할 수 없습니다」
- **원인:** 7일 유효, 이미 사용, 잘못된 토큰
- **해결:** 대표에게 새 초대 링크 요청

### FAQ-B2 · 초대된 이메일과 다릅니다

- **증상:** `invite_email_mismatch`
- **해결:** 초대받은 **동일 이메일**로 가입

### FAQ-B3 · 초대 페이지 회사명 = 내가 입력한 회사명?

- **아니요.** 초대 페이지는 회사명 **입력 없음**. 토큰으로 조회한 **안내 문구**입니다
- **대표가 새 회사 개설:** `/signup` 사용

### FAQ-B4 · 대표가 본인 이메일로 초대 링크를 만들고 가입함

- **결과:** 본인이 직원(MEMBER)이 됨
- **해결:** 대표 본인은 `/signup` CEO 가입. 초대는 **다른 직원** 이메일용

### FAQ-B5 · 예전 링크 `/?invite=토큰`

- **해결:** 새 형식 `/signup/invite?token=...` (구 URL은 리다이렉트되나 새 링크 발급 권장)

### FAQ-B6 · 초대 페이지에서 회사명 확인 실패 (018)

- **증상:** 「018 마이그레이션…」 관련 오류
- **해결:** Supabase SQL Editor에서 `018_invite_preview_rpc.sql` 실행

## C. 브라우저·세션

### FAQ-C1 · `/signup`에서 회사명 입력했는데 직원이 됨 (토큰 잔존)

- **원인:** 브라우저에 **초대 토큰 잔존** + (구버전) 초대 토큰이 CEO 가입보다 우선
- **해결:** `019_ceo_signup_invite_guard.sql` 적용, 시크릿 창에서 `/signup` 재가입

### FAQ-C2 · 구글 로그인 후 직원 가입 화면으로 자동 이동

- **원인:** 이전에 초대 링크를 열어 `sessionStorage`에 토큰 저장
- **해결:** 시크릿 창 + `/signup`에서 CEO 가입 완료

### FAQ-C3 · UI만 「직원」인데 DB role이 비어 있음

- **원인:** `role` null → 화면 기본값 MEMBER
- **해결:** Supabase에서 `profiles.role`, `company_members.role` 확인·수정
